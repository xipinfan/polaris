## 4. 使用建议

1. **Epic -> Story -> Task 层层拆分**，每个 Task 可以直接交给 vibe coding 生成代码。
2. **Task 层包含**：

   * 数据类型定义（shared-types）
   * Core 接口实现
   * Storage Adapter 持久化
   * Console / Extension 页面或组件
   * MCP 工具 / resource 实现
   * 扩展 hook / UI slot 占位
3. **保持版本边界**：V3 只生成 V3 功能，不生成 V4 或未来功能。
4. **迭代顺序建议**：

   * Core 版本、Diff、IssueBundle、ShareService、ProjectSummary
   * Console 页面依次生成（首页、项目页、请求页、Mock、集合页、运行记录、问题现场、版本页、对比页、分享弹层）
   * Extension 快捷入口
   * MCP Server V3 工具和资源
