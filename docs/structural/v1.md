# 《北极星 Epic / Story / Task 结构草案》v0.1

---

## 1. Epic 层（大功能块）

| Epic ID | 名称                | 版本 | 描述                                                                               |
| ------- | ------------------- | ---- | ---------------------------------------------------------------------------------- |
| E1      | Core 基础服务       | V1   | 提供本地代理服务、请求捕获、Saved Request、Mock、Proxy Rule、Storage、MCP 基础工具 |
| E2      | Web Console         | V1   | 提供首页、实时请求页、已保存请求页、Mock 页、调试页、设置页                        |
| E3      | Browser Extension   | V1   | 显示服务状态、代理模式切换、当前站点快捷代理                                       |
| E4      | MCP Server          | V1   | 提供基础 tool / resource 功能                                                      |
| E5      | 项目化管理          | V2   | Project / Environment / Collection / CollectionRun 功能                            |
| E6      | V2 Web Console 扩展 | V2   | 项目页、集合页、运行记录页、环境管理页                                             |
| E7      | V2 MCP 项目级能力   | V2   | Project 级 tools / resources                                                       |
| E8      | 历史版本管理        | V3   | AssetVersion CRUD、查询、对比                                                      |
| E9      | 运行对比            | V3   | CollectionRun 对比、摘要展示                                                       |
| E10     | 问题现场            | V3   | IssueBundle CRUD + 查询 + 快速创建                                                 |
| E11     | 分享功能            | V3   | ShareRecord CRUD + 链接生成、只读分享                                              |
| E12     | 项目摘要            | V3   | ProjectSummary 聚合展示最近失败、问题现场、变更                                    |

## 2. Story 层（用户可感知的功能）

### Epic E1：Core 基础服务

| Story ID | 名称            | 描述                               |
| -------- | --------------- | ---------------------------------- |
| S1.1     | 启动本地代理    | 启动 HTTP/HTTPS 代理服务，捕获请求 |

| S1.3     | Mock 管理       | 创建 Mock，支持一个接口多个方案    |
| S1.4     | Proxy Rule 管理 | 管理代理规则、生成 PAC             |

## 3. Task 层（具体开发任务示例）

### Epic E1 -> S1.2：请求保存

| Task ID | 描述                                 |
| ------- | ------------------------------------ |
| T1.2.1  | 定义 RequestRecord 数据类型          |
| T1.2.2  | 实现 Core 保存请求接口               |
| T1.2.3  | 实现请求保存到 Storage Adapter       |
| T1.2.4  | Console 保存请求弹层开发             |
| T1.2.5  | Console 调用 Core 接口并展示保存结果 |
| T1.2.6  | MCP Server 提供 save_request Tool    |

### Epic E5 -> S5.1：项目 CRUD

| Task ID | 描述                                                    |
| ------- | ------------------------------------------------------- |
| T5.1.1  | Project 数据结构定义                                    |
| T5.1.2  | Core 项目 CRUD 接口实现                                 |
| T5.1.3  | Storage Adapter 持久化实现                              |
| T5.1.4  | Console 项目列表页开发                                  |
| T5.1.5  | Console 项目详情页开发                                  |
| T5.1.6  | MCP Server 提供 list_projects / get_project_detail Tool |

> 其余 Story 可类似生成 Task，将每个功能拆分成数据定义、Core 接口、Storage、Console 页面、MCP 工具调用、扩展 hook 占位等。
