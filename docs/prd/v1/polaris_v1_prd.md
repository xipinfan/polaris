# 《北极星》V1 正式 PRD v0.1

---

## 1. 文档信息

### 1.1 文档名称

《北极星》V1 产品需求文档 PRD v0.1

### 1.2 产品名称

**北极星**

### 1.3 一句话定义

**北极星：一个以本地代理为核心的接口工作台，帮助研发同学更轻松地查看真实请求、保存与重放请求、创建基础 Mock，并通过 MCP 向外部 AI 暴露这些能力。**

### 1.4 版本目标

V1 要先完成一个最小但完整的可用闭环：

* 浏览器代理可快速启停
* 用户可查看真实请求
* 用户可保存请求
* 用户可重放请求
* 用户可创建基础 Mock
* 系统可通过 MCP 对外暴露核心能力
* 架构上预留扩展机制，但不开放第三方插件生态

### 1.5 本期原则

* 不内置 AI
* 不接模型接口
* 不做 AI 对话入口
* 不做复杂自动化测试
* 不做重型抓包工具体验
* 不做团队权限和审批
* 不开放第三方插件安装，但预留扩展机制

# 2. 产品背景

当前研发同学在接口联调、问题排查、请求复现、Mock 构造时，往往需要在多个工具之间切换，例如：

* 代理抓包工具
* 接口调试工具
* Mock 工具
* 请求管理工具
* 文档工具

这类工具虽然专业，但普遍存在一些问题：

* 学习成本高
* 操作链路长
* 请求难以快速沉淀为可复用资产
* 浏览器代理切换不够顺手
* 对 AI / Agent 不友好，缺少标准化本地能力暴露方式
* 页面风格偏“专业工具”，不够通俗

北极星的目标不是重复做一个更复杂的专业工具，而是做一个更容易上手、以常见研发动作驱动的本地接口工作台。

---

# 3. 产品目标与非目标

## 3.1 产品目标

### 目标 1：降低代理使用门槛

通过“本地服务 + 浏览器插件”的方式，让用户快速理解并切换代理状态。

### 目标 2：降低请求查看门槛

让用户用更通俗的方式查看真实请求，而不是一上来面对大量协议细节。

### 目标 3：让请求变成可复用资产

支持把真实请求保存下来，并继续用于重放、调试和 Mock。

### 目标 4：支持最常见的联调 Mock

支持通过固定返回值快速做基础 Mock，满足前后端联调常见场景。

### 目标 5：通过 MCP 对外提供标准能力

将请求查看、保存、重放、Mock 等能力以 MCP 的形式暴露给外部 AI / Agent。

### 目标 6：V1 就确定长期架构

V1 确定好本地服务、浏览器插件、Web 控制台、MCP、扩展机制的边界，保证 V2 / V3 只做增强，不做大改。

---

## 3.2 非目标

V1 明确不做以下内容：

* 内置 AI 助手
* 模型调用能力
* 对话式分析入口
* 自动化测试平台
* 复杂断言系统
* 高级动态 Mock 脚本
* 团队协作与审批流
* 企业权限体系
* 第三方插件市场
* 第三方插件安装机制
* 复杂工作流编排

---

# 4. 目标用户

## 4.1 核心用户

### 前端研发

需要：

* 查看页面真实请求
* 对当前站点快速启用代理
* 创建基础 Mock 进行联调
* 复用抓到的请求

### 后端研发

需要：

* 快速复现请求
* 查看请求参数和返回结果
* 重放和修改请求

### 测试研发 / 联调同学

需要：

* 更简单地查看接口
* 快速做固定返回 Mock
* 保存常用请求用于验证

---

## 4.2 用户特征

* 对 HTTP / API 有基础理解
* 不一定愿意投入时间学习复杂代理工具
* 更关心“现在怎么快速完成动作”
* 希望 UI 简单、直观、通俗
* 可接受安装本地服务和浏览器插件
* 对 MCP / AI 有兴趣，但不希望产品本身复杂化

# 5. 产品定位与设计原则

## 5.1 产品定位

北极星是一个本地运行的接口工作台，核心是：

* 本地代理
* 请求资产化
* Mock
* 浏览器代理控制
* MCP 能力暴露

它不是：

* 内置 AI 产品
* 重型自动化测试平台
* 专业网络调试专家工具
* 第三方插件生态平台

---

## 5.2 设计原则

### 原则 1：通俗优先

页面尽量避免堆砌协议字段和工程术语，优先展示用户最关心的信息和动作。

### 原则 2：渐进增强

先满足最常用动作，再把高级内容放到详情或设置中。

### 原则 3：轻插件、重本地服务

* Browser Extension 负责代理控制和快捷入口
* Local Core 负责核心能力
* Web Console 负责主要工作台体验

### 原则 4：MCP 优先于内置 AI

系统不做模型层，只做工具层和资源层暴露。

### 原则 5：V1 就预留扩展机制

V1 不开放生态，但必须在架构上预留扩展点。

# 7. 核心使用场景

## 场景 1：快速开始抓包

用户安装本地服务与浏览器插件后，通过插件一键启用代理，在控制台中看到当前站点请求。

## 场景 2：快速定位目标请求

用户在请求列表中通过方法、状态码、关键词等条件找到目标请求。

## 场景 3：将真实请求保存下来

用户在请求详情中点击“保存请求”，沉淀为后续可复用资产。

## 场景 4：修改请求参数并重放

用户从已保存请求或实时请求中进入调试页，修改参数后重新发起请求。

## 场景 5：快速构造一个基础 Mock

用户基于某条真实请求创建固定返回 Mock，用于联调或页面开发。

## 场景 6：只对当前站点开启代理

用户不希望全局代理，只希望当前站点流量进入本地代理。

## 场景 7：让外部 AI / Agent 使用本地能力

外部 AI / Agent 通过 MCP 连接北极星，读取请求、查询详情、重放请求、创建 Mock。

# 11. 代理模式需求

## 11.1 模式定义

### Direct

浏览器直连，不走本地代理。

### Global Proxy

浏览器全局走本地代理。

### Rules Proxy

浏览器按规则决定是否走本地代理。

### System

浏览器跟随系统代理。

---

## 11.2 V1 规则能力

支持：

* 按域名规则
* 当前站点快速加入规则
* 基础 bypass

不支持：

* 复杂优先级可视化编辑
* 正则级高级规则编辑
* 跨项目复杂策略

## 11.3 对外文案约束

不使用：

* 单页代理
* 单 tab 代理

统一表达为：

* 对当前站点开启代理
* 仅代理该站点请求

# 15. 核心用户流程

## 15.1 首次使用流程

1. 用户安装北极星本地服务
2. 用户打开 Web 控制台
3. 用户安装浏览器插件
4. 用户通过插件开启代理
5. 用户访问目标页面
6. 用户在实时请求页看到请求

## 15.2 保存请求流程

1. 用户在实时请求中找到目标请求
2. 点击“保存请求”
3. 输入名称
4. 保存成功
5. 在已保存请求页查看

## 15.3 创建 Mock 流程

1. 用户在请求详情点击“创建 Mock”
2. 系统预填 Method / URL / Response
3. 用户确认或调整
4. 保存并启用
5. Mock 生效

## 15.4 重放请求流程

1. 用户在实时请求或已保存请求中点击“重放”
2. 系统发起请求
3. 页面展示响应结果
4. 用户可进一步保存修改版请求

## 15.5 当前站点开启代理流程

1. 用户在目标站点点击插件
2. 插件识别当前 host
3. 用户点击“对当前站点开启代理”
4. 系统切换到规则代理模式并加入规则
5. 当前站点流量进入本地代理

## 15.6 MCP 接入流程

1. 用户在设置页查看 MCP 地址与说明
2. 外部 AI / Agent 连接北极星 MCP Server
3. 外部 AI / Agent 调用工具读取或操作请求资产

# 15A. 扩展机制预留设计

## 15A.1 设计目标

V1 不开放第三方插件生态，但需在 Local Core、MCP 服务层和 Web Console 中预留统一扩展能力，以支持后续对请求处理、Mock 能力、MCP 工具和资产存储的扩展。

## 15A.2 设计原则

* 扩展能力优先放在 Local Core
* Browser Extension 不承担扩展宿主职责
* Web Console 仅预留 UI 扩展位
* V1 仅支持内置扩展，不支持外部安装
* 扩展不能破坏核心主链路稳定性

## 15A.3 扩展宿主位置

### Local Core

作为主要扩展宿主，负责：

* 请求事件扩展
* Mock 事件扩展
* MCP 工具 / 资源注册扩展
* 存储适配扩展

### Web Console

作为 UI 扩展承载层，负责：

* 页面操作位扩展
* 设置入口占位

### Browser Extension

V1 不承载扩展逻辑，只保留固定能力。

# 20. 风险与注意事项

## 风险 1：HTTPS 抓包门槛

需要尽量简化证书安装和说明。

## 风险 2：代理状态理解成本

需要在插件与首页同时清晰展示状态。

## 风险 3：页面过于专业

必须控制字段密度，避免做成传统重型工具风格。

## 风险 4：MCP 接入理解成本

设置页要用通俗语言解释“如何让外部 AI / Agent 连接北极星”。

## 风险 5：扩展机制过早复杂化

V1 只做结构预留，不做开放生态和复杂权限机制。

# 22. V1 一句话总结

**北极星 V1 是一个轻量、通俗、可本地运行的接口工作台，帮助研发快速开启代理、查看真实请求、保存与重放请求、创建基础 Mock，并通过 MCP 向外部 AI 暴露这些能力，同时在架构上预留未来扩展空间。**
