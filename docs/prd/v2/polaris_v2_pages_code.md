

# 《北极星》V2 研发任务拆分稿 v0.1

---

# 1. V2 研发目标与交付边界

## 1.1 V2 必须交付的能力

V2 完成后，用户应可以完成以下升级闭环：

* 创建和管理项目
* 将请求、Mock、代理规则归属到项目
* 创建并切换环境变量
* 将请求加入集合
* 运行请求集合
* 查看运行记录
* 在项目维度管理接口资产
* 通过 MCP 向外部 AI / Agent 暴露项目级上下文

---

## 1.2 V2 不纳入本轮开发的能力

以下内容明确不纳入 V2：

* 历史版本
* 问题现场
* 分享能力
* 运行对比
* 轻协作
* 内置 AI
* 第三方插件市场
* 第三方扩展安装
* 企业权限体系
* 重型自动化测试平台

---

# 2. V2 技术模块拆分

V2 仍然沿用 V1 的 4 个模块，但需要增强：

## 模块 A：Local Core（增强）

新增：

* Project Service
* Environment Service
* Collection Service
* Run History Service

## 模块 B：Web Console（增强）

新增：

* 项目列表页
* 项目详情页
* 请求资产页（项目化）
* 集合详情页
* 运行记录页
* 环境管理模块

## 模块 C：Browser Extension（轻增强）

新增：

* 当前活跃项目展示
* 当前站点加入当前项目规则

## 模块 D：Shared Contract / Schema（增强）

新增：

* Project
* Environment
* RequestCollection
* CollectionRun
* 项目级 MCP tools / resources 协议

# 3. 模块 A：Local Core 研发任务拆分

---

## A-1 Project Service（项目服务）

### 目标

建立项目模型，让请求、Mock、规则等对象有归属。

### 任务

* A-1-1 Project 数据结构定义
* A-1-2 项目创建接口
* A-1-3 项目列表接口
* A-1-4 项目详情接口
* A-1-5 项目更新接口
* A-1-6 项目删除接口
* A-1-7 默认项目设置接口
* A-1-8 最近使用项目记录接口

### 产出

* 项目可 CRUD
* 系统可识别默认项目 / 最近项目

---

## A-2 Saved Request 升级（项目化）

### 目标

让保存请求可以归属到某个项目。

### 任务

* A-2-1 SavedRequest 结构增加 projectId
* A-2-2 保存请求时支持写入 projectId
* A-2-3 按项目查询请求列表
* A-2-4 请求详情返回所属项目
* A-2-5 请求移动项目能力（可选后置）
* A-2-6 请求筛选接口支持 projectId

### 产出

* 请求资产具备项目归属

---

## A-3 Mock 升级（项目化）

### 目标

让 Mock 规则支持项目归属。

### 任务

* A-3-1 MockRule 结构增加 projectId
* A-3-2 Mock 创建 / 更新支持 projectId
* A-3-3 按项目查询 Mock
* A-3-4 Mock 列表接口支持项目筛选
* A-3-5 Mock 复制时继承项目归属

### 产出

* Mock 资产具备项目归属

---

## A-4 Proxy Rule 升级（项目化）

### 目标

让代理规则支持项目归属和项目级管理。

### 任务

* A-4-1 ProxyRule 结构增加 projectId
* A-4-2 当前站点加规则时支持绑定项目
* A-4-3 按项目查询规则列表
* A-4-4 当前项目默认规则策略接口
* A-4-5 规则优先级字段预留

### 产出

* 代理规则具备项目归属
* 当前站点可写入到某项目规则中

---

## A-5 Environment Service（环境变量服务）

### 目标

建立项目级环境变量能力。

### 任务

* A-5-1 Environment 数据结构定义
* A-5-2 环境创建接口
* A-5-3 环境列表接口
* A-5-4 环境详情接口
* A-5-5 环境更新接口
* A-5-6 环境删除接口
* A-5-7 默认环境设置接口
* A-5-8 请求执行时变量替换逻辑
* A-5-9 请求执行时无默认环境的友好错误返回

### 产出

* 环境变量可 CRUD
* 请求执行支持基础变量替换

### V2 范围约束

仅支持简单键值替换，不做：

* 表达式解析
* 脚本变量
* 链式依赖变量

---

## A-6 Collection Service（请求集合服务）

### 目标

支持把请求组织成集合，并执行集合。

### 任务

* A-6-1 RequestCollection 数据结构定义
* A-6-2 集合创建接口
* A-6-3 集合列表接口
* A-6-4 集合详情接口
* A-6-5 集合更新接口
* A-6-6 集合删除接口
* A-6-7 请求加入集合接口
* A-6-8 请求移出集合接口
* A-6-9 集合内顺序调整接口

### 产出

* 集合可被创建和维护
* 请求可加入集合

---

## A-7 Collection Runner（集合执行）

### 目标

支持批量运行集合中的请求。

### 任务

* A-7-1 CollectionRun 数据结构定义
* A-7-2 集合运行接口
* A-7-3 集合逐条执行逻辑
* A-7-4 单条执行结果记录
* A-7-5 整体运行状态汇总
* A-7-6 集合运行失败处理逻辑
* A-7-7 默认环境注入执行过程

### 产出

* 集合可运行
* 每次运行有结果记录

### V2 范围约束

先做顺序执行，不做：

* 并发编排
* 条件分支
* 前后置脚本

---

## A-8 Run History Service（运行记录服务）

### 目标

承接集合运行结果的持久化与查询。

### 任务

* A-8-1 CollectionRun 存储实现
* A-8-2 运行记录列表接口
* A-8-3 运行记录详情接口
* A-8-4 按项目查询运行记录
* A-8-5 按状态筛选运行记录
* A-8-6 最近运行摘要接口

### 产出

* 运行记录可查询
* 首页 / 项目页可读取最近运行

---

## A-9 MCP Server 升级（项目级）

### 目标

从 V1 的单请求能力扩展到项目级能力。

### 任务

* A-9-1 增加项目级 tool 注册
* A-9-2 增加项目级 resource 注册
* A-9-3 实现新增 tools：

  * list_projects
  * get_project_detail
  * list_saved_requests
  * list_mock_rules
  * run_collection
  * get_project_env
  * list_proxy_rules
* A-9-4 实现新增 resources：

  * 项目列表
  * 项目详情
  * 项目请求列表
  * 项目 Mock 列表
  * 项目环境列表
  * 项目运行摘要
* A-9-5 MCP tool / resource 参数校验增强

### 产出

* 外部 AI / Agent 可围绕项目工作，而不只是操作单条请求

---

## A-10 Storage Adapter 升级

把 V2 新对象纳入统一存储适配层。

# 11. 研发风险点

---

## 风险 1：项目化之后页面会变重

要通过：

* 卡片化
* 动作化文案
* 二级入口而非多级后台菜单
  来控制复杂度。

---

## 风险 2：环境变量功能容易膨胀

必须守住：

* 简单键值替换
* 不做表达式
* 不做脚本

---

## 风险 3：集合功能容易演变成测试平台

V2 只做：

* 组织请求
* 批量运行
* 查看结果

不做：

* 编排系统
* 断言 DSL
* 脚本链路

---

## 风险 4：MCP 能力膨胀

V2 的 MCP 只围绕项目资产暴露，不引入复杂 workflow orchestration。

---

## 风险 5：插件承担过多项目逻辑

插件只做轻上下文感知，不做复杂编辑能力。

---

# 13. 一句话研发版本定义

**北极星 V2 的研发目标是把 V1 的单次请求工具升级为项目级接口资产工作台，让请求、Mock、规则、环境和集合都能围绕项目持续管理，并通过 MCP 向外部 AI / Agent 暴露项目级能力。**
