# 浏览器扩展使用说明

Polaris 浏览器扩展是一个轻量控制面板。

它不承载复杂业务，而是负责帮助你更快完成这些动作：

- 查看 Core 是否在线
- 切换代理模式
- 将当前站点加入或移出代理规则
- 打开 Console 和设置页

## 构建扩展

```bash
pnpm --filter @polaris/extension build
```

构建产物目录：

- [apps/extension/dist](/E:/code/polaris/apps/extension/dist)

## 加载扩展

以 Chrome 或 Edge 为例：

1. 打开扩展管理页面
2. 开启开发者模式
3. 选择“加载已解压的扩展程序”
4. 选择 `apps/extension/dist`

## 扩展能做什么

### 1. 查看 Polaris 是否在线

扩展会读取 Core 服务状态，并显示当前是否在线。

### 2. 切换代理模式

当前支持的代理模式：

- `direct`
- `global`
- `rules`
- `system`

如果你只是想针对当前站点做联调，通常更推荐使用 `rules`，而不是 `global`。

### 3. 快速代理当前站点

扩展可以把当前标签页对应的站点快速加入代理规则，也可以一键移除。

这适合日常联调场景：

1. 打开目标站点
2. 打开扩展弹窗
3. 切到 `rules`
4. 选择“仅代理当前站点”
5. 回到 Console 查看 Traffic

### 4. 快速打开控制台

扩展可以直接打开：

- Console 首页
- Settings 页面

## 端口兼容

扩展不再假定 Core 一定运行在 `9001`。

它会自动探测一组本地 API 端口，并在成功连接后缓存结果。因此在端口被占用、Core 自动切换端口的情况下，扩展仍然可以工作。

扩展也会尝试发现当前 Console 地址，而不是只固定打开 `5173`。

## 常见问题

### 扩展里显示 Core 离线

优先检查：

- `pnpm polaris:start` 是否已经执行
- 扩展是否重新构建并重新加载
- `pnpm polaris:status` 输出的健康检查地址是否可访问

### 为什么我切了规则模式但没有抓到请求

优先检查：

- 当前站点是否已经加入规则
- 页面是否已经刷新
- 浏览器代理是否已经被成功应用

### 扩展适合替代 Console 吗

不适合。

扩展更适合做快捷控制，而不是完整操作。真正查看请求详情、保存请求、调试请求、管理 Mock，还是更适合在 Console 中完成。
