# Web Console 使用说明

Web Console 是 Polaris 的主要操作界面。

如果你平时主要通过浏览器查看请求、保存请求、管理 Mock 和切换设置，这份文档就是给你看的。

## 启动方式

开发模式下：

```bash
pnpm dev
```

然后访问：

- [http://127.0.0.1:5173](http://127.0.0.1:5173)

Console 会自动发现当前 Polaris Core API 地址，因此即使默认 API 端口被占用，页面也仍然可以找到服务。

## 主要页面

### 首页

路径：

```text
/
```

这里主要用于快速了解当前状态：

- Core 是否在线
- 当前代理模式
- 最近请求数量
- Mock 数量
- MCP 是否开启

首页会优先使用本地缓存快速显示上一次成功的数据，然后在后台静默刷新。

### Traffic

路径：

```text
/traffic
```

这是最核心的页面，适合做下面这些事情：

- 查看实时请求
- 过滤请求
- 查看请求详情
- 重放请求
- 复制 curl
- 将请求带入调试页
- 从请求创建 Mock

请求页中的证书状态也会优先使用缓存，再静默刷新。

### Requests

路径：

```text
/requests
```

适合管理已经保存下来的请求资产。

常见操作：

- 查看已保存请求
- 编辑请求
- 重放请求
- 带入调试
- 转成 Mock

### Mock

路径：

```text
/mock
```

适合管理本地 Mock 规则。

常见操作：

- 创建 Mock
- 编辑 Mock
- 启用或停用 Mock
- 查看不同规则的状态

### Debug

路径：

```text
/debug
```

这是一个手动请求调试页，适合：

- 直接发起请求
- 检查返回结果
- 验证接口状态
- 保存调试请求

### Settings

路径：

```text
/settings
```

这里主要展示：

- 当前代理端口
- API 地址
- MCP 地址
- 代理模式
- 证书状态
- MCP 开关状态

Settings 页面同样会优先使用本地缓存，然后进行后台刷新。

## 证书下载

在 Settings 页面和 Traffic 页面中，都可以看到证书相关入口。

证书下载地址会根据当前实际 API 端口生成，而不是固定写死某个端口，因此在端口自动切换时仍然可用。

## 端口被占用时会发生什么

Polaris 默认优先使用：

- API：`9001`
- Console：`5173`

如果 `9001` 被占用，Core 会自动切换到新的端口。Console 不需要你手动改配置，它会自动探测并连接到新的 API 地址。

## 常见问题

### 页面打开了，但一直显示连接中

优先检查：

- Polaris 是否已经启动
- `pnpm polaris:status` 返回的健康检查地址是否可访问

### 为什么第二次打开页面更快

因为首页、Settings 和 Traffic 的部分状态会先从本地缓存中读取，再后台静默更新。

### Console 默认适合谁

如果你更习惯可视化操作，而不是直接通过 MCP 或命令行调试，Console 会是最合适的入口。
