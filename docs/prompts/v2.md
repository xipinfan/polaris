你是一名全栈开发助理，任务是生成北极星（Polaris）项目 V2 的代码。

【项目背景】
北极星 V2 是在 V1 基础上升级的项目级接口资产工作台。  
主要目标是：
- 支持“项目”概念，将请求、Mock、规则、环境、集合等资产归属到项目
- 支持环境变量（基础键值替换）
- 支持请求集合管理及执行
- 支持集合运行记录和最近运行摘要
- MCP 提供项目级工具和资源
- Web Console、Browser Extension、Local Core 的目录结构沿用 V1
- UI 仍保持通俗、轻量、易上手
- 不内置 AI，不开放第三方插件市场

V2 不包含：
- 历史版本管理
- 运行对比
- 问题现场
- 分享能力
- V3 功能

---

【技术边界与架构】
本项目沿用 V1 monorepo：

polaris/
  apps/
    core/        # Local Core
    console/     # Web Console
    extension/   # 浏览器插件
  packages/
    shared-types/
    shared-contracts/
    mcp-contracts/
    shared-utils/
    extension-sdk/
  docs/
  scripts/
  configs/

模块职责：

1. Core：
   - Project Service（项目 CRUD）
   - Environment Service（环境变量 CRUD）
   - RequestCollection Service（请求集合 CRUD + 批量运行）
   - CollectionRun Service（运行记录 CRUD）
   - Request / Mock / Proxy Rule 支持项目归属
   - MCP Server 提供项目级 tools / resources
   - Storage Adapter
   - 扩展宿主预留

2. Console：
   - 首页（最近项目 / 最近运行）
   - 项目列表页
   - 项目详情页（项目摘要、最近请求、最近运行、环境、规则）
   - 请求资产页（项目化请求管理）
   - Mock 页（项目化管理）
   - 集合详情页（编辑、运行）
   - 运行记录页（列表 + 详情）
   - 环境管理页
   - 设置页（默认项目/默认环境、MCP 说明）

3. Extension：
   - 显示当前活跃项目
   - 当前站点加入当前项目规则
   - 最近项目快捷入口（可选）
   - 保持轻量原则

4. Shared-types / contracts：
   - Project、Environment、RequestCollection、CollectionRun
   - 枚举、常量、状态
   - Core / Console / Extension 统一使用
   - MCP 项目级 tools / resources schema
