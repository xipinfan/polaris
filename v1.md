【V1 功能范围】
- Core（apps/core/）：
  - 启动本地代理（HTTP/HTTPS）
  - 捕获请求并实时展示
  - 保存请求 CRUD
  - Mock 管理（一个接口多方案）
  - Proxy Rule / PAC 管理
  - Storage Adapter
  - MCP Server（V1 基础 tools / resources）
  - 扩展宿主预留 hook / UI slot

- Web Console（apps/console/）：
  - 首页
  - 实时请求页（请求列表、详情、保存、重放、创建 Mock）
  - 已保存请求页
  - Mock 页
  - 调试页
  - 设置页

- Browser Extension（apps/extension/）：
  - 显示 Core 服务状态
  - 代理模式切换
  - 当前站点快速加入规则

- MCP Server（packages/mcp-contracts/）：
  - list_requests
  - get_request_detail
  - save_request
  - replay_request
  - create_mock_rule
  - enable_mock_rule
  - run_request
  - list_proxy_rules

> 注意：V1 不包含项目、集合、运行记录、历史版本、对比、问题现场或分享功能。

---

【目录约束】
严格遵循 `docs/architecture/polaris_monorepo.md` 中的 monorepo 结构：

```

apps/
core/
console/
extension/
packages/
shared-types/
shared-contracts/
mcp-contracts/
shared-utils/
extension-sdk/
docs/
scripts/
configs/

```

- Core 内模块按业务能力拆分（modules/requests、modules/mock、modules/proxy、modules/mcp、modules/storage）
- Console 按 pages + features 拆分
- Extension 保持轻量，仅展示与控制，不处理业务逻辑
- 所有对象类型必须使用 `shared-types/`
- Core 与 Console / Extension 通信必须使用 `shared-contracts/`
- MCP 工具与资源必须使用 `mcp-contracts/`

---

【生成要求】
1. 严格只生成 V1 功能
2. 所有文件必须放在对应 monorepo 路径
3. 保留扩展 hook / UI slot 占位
4. 文件命名使用小写驼峰或 kebab-case
5. 页面和组件代码必须可直接渲染
6. Core / Console / Extension / MCP 功能必须可独立运行
7. 不生成 V2 / V3 功能
8. 保持 UI 简洁、通俗易上手
9. 数据类型统一使用 shared-types
10. Console / Extension 调用 Core 接口必须使用 shared-contracts
